# Hypothesis Tracker — 新建假设

创建一个新的投资假设文件，引导用户填写核心逻辑、证伪条件和初始确定性。

---

## 元数据

| 字段 | 值 |
|------|-----|
| **触发命令** | `/ht-new` |
| **兼容别名** | `/ht-create`, `/hypothesis-new` |
| **触发词** | "新假设"、"new hypothesis"、"create hypothesis" |
| **版本** | v1.1 |
| **依赖文件** | templates/hypothesis-tracker-hypothesis-template.md, hypothesis/ |

---

## 执行步骤

### Step 1: 收集信息

向用户询问以下信息（可以从自然语言中提取，不足的部分追问）：

1. **假设名称**：简短描述核心主题（如"AI 算力持续增长"、"深水钻井复苏"）
2. **核心逻辑**：1-3 句话说明为什么相信这个假设（因为 X → 所以 Y）
3. **证伪条件**：至少 2 个，每个必须是"可观测指标 + 具体阈值 + 时间窗口"三元组。
   - ❌ "基本面恶化" — 太模糊
   - ✅ "Q2 续费率低于 60%" — 指标+阈值，窗口是本季度
   - 如果用户给的是模糊条件，帮他细化成三元组再确认。
4. **投资方向**：打算投什么类型的标的？
5. **初始确定性**：0-100%，50% 表示完全不确定
6. **是否需要子假设拆解**（可选）：如果这个假设有多个可能情景、概率不同，可以拆成子假设（如 H2a 30% / H2b 50% / H2c 20%）。单一路径的假设跳过。
7. **概率校准**（可选，仅当有 base rate 参照时）：如果用户的个人信心和历史 base rate 差距较大，记录两者并给出折中值，说明为何偏向某一方。

如用户只给了一句话（如"我觉得 AI 算力会继续涨"），帮助提炼结构化信息并确认。

### Step 2: 确定 ID

扫描 `hypothesis/` 目录下已有的 H*.md 文件，找到最大编号，+1 作为新 ID。

如果 `hypothesis/` 目录不存在，创建它，ID 从 H1 开始。

### Step 3: 生成文件

1. 读取 `templates/hypothesis-tracker-hypothesis-template.md`
2. 替换所有 `{PLACEHOLDER}` 为用户输入：
   - 证伪条件：填入指标/阈值/窗口三列
   - 子假设拆解：如果用户没选这个，删掉整个"子假设拆解"区块（含标题和表格）
   - 概率校准说明：没有则删掉该段
3. `{DATE}` 使用当天日期（YYYY-MM-DD）
4. 写入 `hypothesis/H{N}-{slug}.md`
   - slug：假设名称的简短中文或英文版本，用连字符连接

### Step 4: 关联检查

如果已有其他假设文件，快速扫描标题和投资方向：
- 发现可能的关联 → 提示用户："这个假设和 H{n}（{名称}）可能有关联，需要在两边加交叉引用吗？"
- 无关联 → 跳过

### Step 5: 输出确认

```
✅ 已创建假设：H{N} {名称}
📄 文件：hypothesis/H{N}-{slug}.md
📊 初始确定性：{X}%
🔴 证伪条件：{数量} 个
🌿 子假设分支：{数量}（若有）

下一步：
- 开始收集证据，添加到"📰 证据时间线"
- 每月回顾一次 Kill Thesis（模板已预留区块）
- 发现相关交易机会时，用 /ht-trade 记录
- 用 /ht-status 查看所有假设的状态
```
