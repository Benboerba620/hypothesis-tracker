# Investment Rules

> 从假设验证和反复出现的模式中提炼的投资规则。
> 假设中某个模式被确认 3 次以上 → 提议添加为 rule。
> 新证据与 rule 矛盾 → 移到 "Rules Under Review"，在下次周度回顾讨论。

## Trading System

> 核心理念：**买认知不买价格。Thesis 驱动方向，仓位匹配信念，纪律管住风险。**
> 对的时候赚够，错的时候亏小。集中下注高确定性的 thesis，快速认错低确定性的仓位。

### 组合约束

| 约束 | 上限 | 说明 |
|---|---|---|
| 单票 | 30% NAV | 需有止损执行记录支撑 |
| 单 thesis | 50% NAV | 超级主题适用 |
| 持仓数量 | ≤8 个 | 每新建仓问"它值不值得挤掉最弱的" |
| 最低现金 | 10% | 现金是弹药，不是浪费 |

> 以上数值可在 `hypothesis-tracker.yaml` 的 `risk_rules` 中调整。

### Layer 1: 建仓——"买多少"

仓位匹配理解深度，建仓时一次买到位，不留"以后再加"的模糊空间。

| 认知阶段 | 仓位 | 必须写清楚 |
|----------|------|-----------|
| 逻辑不清楚 | 不买 | — |
| 大逻辑清楚，验证链不够 | 观察仓 1-3% | "哪些环节还没验证" |
| 逻辑 + 验证清楚，等催化剂 | 核心仓 5-10% | "等什么数据，确认后补多少" |
| R1 三问都能答 | 重仓 15-25% | 加仓计划（触发条件 + 仓位） |

### Layer 2: 加仓——"什么时候加"

**唯一理由：thesis 确定性提高了。**

每次加仓必须能回答："上次买完之后，发生了什么新的事让我更确信了？"

| 规则 | 内容 |
|------|------|
| 新证据原则 | 每次加仓必须有新证据。价格上涨本身不是理由 |
| 递减仓位 | 每次加的量 ≤ 上一次（金字塔形） |
| 最多 3 次 | 含建仓，最多 3 次。用完只持有 |
| 加完上移止损 | 至少保住前批次本金 |
| 禁止摊低亏损 | 浮亏时默认不加。问自己：没有仓位的话今天会买吗？ |

### Layer 3: 止损——"什么时候跑"

| 类型 | 触发 | 动作 |
|------|------|------|
| **Thesis 止损** | 假设被证伪 | 不看价格，直接走 |
| **价格止损** | 跌到预设止损价 | 先减半 + 48h 冷静再决策 |

### Layer 4: 卖出——"赚了什么时候走"

| 阶段 | 规则 |
|------|------|
| 目标价以下 | 只看 thesis，不看价格。不提前跑 |
| 目标价以上 | 启用移动止盈，触发减半 |
| Thesis 完全兑现 | 清仓 |
